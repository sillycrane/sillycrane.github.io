发表于 2019-04-12 | 分类于 scm设计

START

正式开始写这个系列，写这个文章的时候，scm-设备端已经写了一大半了，UI已经设计好了，数据库也差不多了，就是在纠结于图片存入sqlserver用什么方式存。

4-12-12：40更新

process

  • ui设计好了
  • 设备端登录做好了
  • baidu_api类封装好了
  • cam类也封装好了（调用摄像头）

problem

  • 早上用base64编码之后存进去，想的是这样做的话就可以直接传编码过去给api比较了，但是一张摄像头拍的照片确实太大了，会让数据库变得特别巨大，base64编码之后也太长了，读取也不方便
  • 另外就是直接图片单独开一个文件夹，然后直接在数据库存储文件名，文件的话用学生的id来命名

    solution

  • 最后还是决定采用第二种方案，牺牲一点灵活性，换取一点容量
  • 学生表需要再完善一点
  • 增加表操作，每个班级单独开一个简单的表

4-12-14：00更新

process

  • ui设计好了
  • 设备端登录做好了

  • 导入课程列表也做好了
  • 根据课程导出学生也做好了

    problem

  • 调用百度API是在线的识别，需要网络，并且需要3秒左右，时间有点太长了，学姐给我说有个虹软的离线sdk可以提高速度
  • 看了看虹软的人脸识别SDK，它支持C/C++，我用的C#。。。难受

    solution

  • 两种方案
  • 一是百度一手，看看C#怎么调用C++代码
  • 二是C++生成EXE可执行文件，传参返回调用
