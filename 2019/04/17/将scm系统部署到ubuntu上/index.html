将scm系统部署到ubuntu上

发表于 2019-04-17 | 分类于 scm设计

start

昨天突然想把scm管理系统客户端上传到我买的阿里云服务器上，遇到了种种问题，好在最后都解决了，成功的上传了scm管理系统客户端。下面我就来记录一下整个过程，想着都是泪啊。

yesterday

先试试水

  • 确定了使用lamp组合(linux+apache2+mssql代替mysql+python代替php)

    django的部署

  • 服务器上安装django
  • 修改apache2配置并重启服务
  • 上传scm客户端，并添加允许host
  • 运行scm客户端
  • 远程访问成功

    mssql的安装

  • 安装sqlser真的是有感而发啊
  • 尽量去看官方的方法吧，随便百度一些野方法太容易出错了
  • 根据官方的方法安装mssql
  • 备份本地的数据库，上传到服务器
  • 恢复备份到服务器数据库

    爆炸

  • 不知道是哪里配置错了(应该是apache2配置错了)
  • apache2能运行，但是不能访问
  • 气到爆炸
  • 睡觉吧

today

try

  • 一大早起床研究apache2配置结构，一个一个的修改
  • emmmmmmmmm MTM还是不行

    solution

  • 最后只有祭出我的大招
  • 回滚之术emmmm看了看快照。。。。
  • 牛皮，没得快照。。。。沉默。。。。
  • 好吧，重置系统
  • 按照昨天的步骤再走了一次
  • 有了经验，这次就快多了，不手贱乱修改
  • 成功的在服务器上部署了scm客户端
