命令执行漏洞

system()会直接回显结果打印输出
exec()、shell_exec()、passthru()返回执行结果，但需要单独打印
popen()、proc_open()不会直接返回执行结果，而是返回一个文件指针，但命令已经执行
反引号（`）实际上就是调用了shell__exec()函数

变量覆盖漏洞

extract()

从数组中将变量导入到当前的符号表
<?php
$b=3;
$a=array(‘b’=>’1’);
extract()$a);
print_r($b);
?>
输出1

parse_str()

解析字符串并且注册成变量
<?php
$b=1;
parse_str(‘b=2’);
print_r($b);
?>
输出2

import_request_variables()

将GET、POST、COOKIE的参数注册成变量，调用该函数相当于开启了全局变量注册，php5.4之后该函数被取消（参数中G代表GET，P代表POST，C代表COOKIE）

<?php
$b=1;
import_request_variables(‘GP’);
print_r($b);
?>

访问1.php?b=2输出2

$$变量覆盖

<?php
$a=1;
foreach(array(‘_GET’,’_COOKIE’,’_POST’) as $_request){
foreach($$_request as $_key=>$_value){
echo $_key.’##’;
$$_key=addslashes($_value);
}
}
echo $a;
?>

访问1.php?a=3输出a##3

逻辑处理漏洞

in_array()

判断一个之是否在某个数组列表里面，但在比较之前会自动做类型转换，即in_array(‘1qew’,array(1,2,3))是正确的

is_numeric()

判断一个变量是否为数字，但这个函数存在一个问题，当传入的参数为hex时则直接通过并返回true，而mysql是可以直接使用hex编码代替字符串明文的

弱比较

双等于（==）和三等于（===），但双等于会在判断前做变量类型转换，而三等于不会，即2aaa==2是true，2aaa===2是false

水平越权

指相同等级权限的用户之间

垂直越权

指不同的权限等级的用户之间

GPC的空子
